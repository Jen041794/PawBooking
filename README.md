# 🐾 毛孩預約 PawBooking

寵物美容／洗澡的線上預約系統（純前端練習專案）。使用者可以選服務、挑日期時段、填寫寵物與飼主資料、送出預約，並在「我的預約」管理或取消。

> 純前端專案，資料以瀏覽器 `localStorage` 模擬後端儲存，重新整理不會遺失。

## 技術棧

- **Vite + React 19** — 前端框架與開發環境
- **React Router** — 多頁面預約流程
- **React Bootstrap + Bootstrap 5（SCSS 客製）** — UI 與配色（覆寫 SCSS 變數做健康自然系主題）
- **React Context** — 跨頁面共享預約草稿與清單
- **date-fns** — 日期與時段計算
- **localStorage** — 預約資料儲存

## 預約流程

```
選服務 → 選日期/時段 → 填資料 → 確認 → 完成 → 我的預約（可取消）
```

## QA 邊界測試清單

預約系統的價值在於把「容易出錯的狀態」處理對，以下是本專案目前已實作並可驗證的邊界：

- ⛔ 不能預約過去的日期
- ⛔ 不接受當天預約，最早只能預約明天（日期選擇器 + 時段邏輯雙重把關）
- ⛔ 同一時段不能被重複預約（時間區間重疊判斷）
- ⛔ 全套美容 120 分鐘會佔用連續多個時段，後續被佔走的時段不可選
- ⛔ 最後可預約的時段為 20:00，晚於此時間不可預約
- ⛔ 表單必填欄位驗證（寵物名、飼主姓名、電話）
- ⛔ 手機號碼格式驗證（09 開頭共 10 碼）
- ✅ 取消預約後，該時段會釋放回去、可再次預約
- ✅ 切換服務時會清掉已選的日期/時段，避免時長對不上

## 本機執行

```bash
npm install
npm run dev
```

## 測試

時段衝突與邊界邏輯（`src/utils/booking.js`）有完整的單元測試（Vitest），涵蓋過去日期、當天不可約、晚於最後預約時間、時段重疊、背靠背接續、120 分鐘服務佔多時段等情境。

```bash
npm run test       # watch 模式
npm run test:run   # 跑一次（CI 用）
```
